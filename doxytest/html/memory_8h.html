Detailed Description

Provides the memory related Phasor scripting functions.

Note: As of 01.00.10.059 all read/write functions have been overloaded to optionally remove the need for the address offset. For such cases you shouldn't include address_offset and the address used will be exactly what's specified and not (base_address + address_offset) as in previous versions.

For example, the below are equivalent

readbyte(12345, 5)
readbyte(12350)

DEFINITIONS

byte - 8 bit positive integer from [0,255]

char - 8 bit integer from [-128, 127]

word - 16 bit positive integer from [0,65,535]

short - 16 bit integer from [-32768, 32767]

dword - 32 bit positive integer from [0, 4,294,967,295]

int - 32 bit integer from [-2147483648, 2147483647]

float - 32 bit floating point number from [-3.402823466e+38, 3.402823466e+38]

void l_readbit ( PHASOR_API_ARGS  )

Reads a bit from the specified memory address.

Parameters
base_addressThe base address to use.
[address_offset]The offset relative to base_address.
bit_offsetWhich bit to read (starting at 0)
Returns
The specified bit (ie 1 or 0)

Example usage:

local b = readbit(0x12345678, 3, 2) -- read the 3rd bit at 0x12345678 + 3
local b = readbit(0x12345678, 2) -- read the 3rd bit at 0x12345678
